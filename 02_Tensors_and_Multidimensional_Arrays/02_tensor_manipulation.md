# 🧠 Tensors & Data Representation in PyTorch (L2 | CO1)

---

## 🔹 1. What is PyTorch?

PyTorch is a Python-based library used for building deep learning models.

### ✅ Key Features

* Open-source
* Built on Torch library
* Developed by Facebook AI Research (FAIR)

### 🔸 Capabilities

* Build neural networks
* Train models efficiently
* Handle large-scale computations

---

## 🔹 2. Role of Tensors in PyTorch

Tensors are the **fundamental data structure** in PyTorch.

### ✔ Definition

A tensor is:

* A generalization of scalars, vectors, and matrices
* A multidimensional array

---

## 🔹 3. Why Tensors are Needed in Deep Learning

### 🧩 Core Idea

Deep Learning = **Transforming input representation → output representation**

Tensors are used to:

* Store input data
* Perform computations
* Represent outputs

---

## 🔹 4. Data Transformation in Deep Learning

### 🔸 Concept

A deep learning system:

* Takes input data
* Transforms it into output predictions

### 🔸 How it works

* Learns patterns from multiple examples
* Extracts common features

---

## 🔹 5. Role of Floating-Point Numbers

Neural networks operate using **floating-point numbers (decimals)**.

### ✔ Why?

Real-world data must be:

* Encoded → Processed → Decoded

### 🔴 Key Conclusion

👉 All computations in deep learning are performed using **floating-point numbers stored in tensors**

---

## 🔹 6. Multi-Stage Transformation (Neural Networks)

Deep neural networks process data in **multiple layers**.

### 🔸 Intermediate Representations

At each layer:

* Data is transformed into a new representation

👉 Called: **Intermediate Representations**

---

## 🔹 7. Understanding Intermediate Representations

### ✔ Properties

* Generated by combining input with neuron weights
* Each layer produces a unique representation

### ✔ Example (Image Recognition)

* Early layers → edges, textures
* Deeper layers → complex features (eyes, objects)

---

## 🔹 8. Key Insight

Intermediate representations:

* Are collections of floating-point numbers
* Capture meaningful features of input data

---

## 🔹 9. Full Data Flow in Neural Networks

### 🔄 Pipeline

**Input → Intermediate Representations → Output**

* Input: Image → pixel values (integers)
* Converted to: Floating-point tensors
* Output: Probabilities (e.g., classes like sun, sea)

### 🔴 Important Rule

👉 Similar inputs → Similar intermediate representations (especially in deeper layers)

---

## 🔹 10. What is a Tensor? (Deep View)

A tensor is an **N-dimensional array**.

---

## 🔹 11. Types of Tensors

| Type   | Dimension | Example            |
| ------ | --------- | ------------------ |
| Scalar | 0D        | 5                  |
| Vector | 1D        | [1,2,3]            |
| Matrix | 2D        | [[1,2],[3,4]]      |
| Tensor | nD        | Higher dimensional |

### ✔ General Rule

N-D tensor → requires N indices

---

## 🔹 12. Dimensionality of Tensors

Dimensionality = number of axes (indices)

Example:

* 3D tensor → needs 3 indices

---

## 🔹 13. PyTorch vs NumPy

### ✔ Similarities

* Support multidimensional arrays

### ❗ Differences

* PyTorch supports GPU acceleration
* PyTorch supports automatic differentiation (Autograd)

## 🔹 14. Tensor vs Traditional Arrays
